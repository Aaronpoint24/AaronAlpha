# ソリッドモードの同期不備に関する修正報告

## 現象
ガーベージマット（GM）の外側に、以前の編集内容の残骸が不自然に残る。

## 原因
Solidモード専用の `solid_buffer` や `c1s_buffer` が、GMパラメータ変更時に範囲外のクリアを適切に行っていない。

## 対策
- Rust側の update_solid_params() において、GMの外側境界（Top, Bottom, Left, Right）を厳格にチェックし、範囲外のバッファを強制的に 0 または適切な初期値でクリアするようにロジックを修正。
- 描画レイヤーの合成順序を整理し、GMによるマスクが最優先されるようにした。
