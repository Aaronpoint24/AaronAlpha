## 1. 概要
- **目的**: 機能全般の整合性、バグ、翻訳の不備、UIの矛盾の洗い出し
- **対象ファイル**:

    - js/RenderEngine.js
    - js/i18n.js

## 2. 検証項目

### A. i18n（国際化）の整合性
- [ ] index.html 内の全テキスト要素に data-i18n 属性がついているか
- [ ] data-i18n のキーが i18n.js に存在するか
- [ ] 動的に変更されるテキスト（ボタンのON/OFFなど）が正しく翻訳されているか

### B. UI/UX の動作矛盾
- [ ] **背景設定の同期**: Basic/Trash/Solid 間でスイッチの状態が正しく同期されるか
- [ ] **モード遷移時の初期化**:
    - Trashモード初回遷移時の「黒背景強制」とスイッチの整合性
    - Solidモード遷移時の初期状態
- [ ] **比較モード（Comparison Mode）**:
    - ON時に操作ロックされるべき要素（ドロップ、Lasso、調整モード）が正しくガードされているか
- [ ] **ズーム & リセット**:
    - スペース+右クリック、ホイールダブルクリック双方が fitToScreen() (100%) を呼んでいるか

### C. ロジックの正確性
- [ ] **Solidモード**: 
    - raycastDist パラメータが正しく反映されているか
    - プレビューボタンのトグル動作と描画ロジックの整合性
- [ ] **Trashモード**:
    - ガーベージマット編集と位置調整モードが排他制御されているか
    - プレビューボタンのトグル動作
