# Git履歴のクリーンアップ計画（ソース完全隠蔽）

アーロン様、ご質問ありがとうございます。
現状を確認いたしましたところ、**残念ながら過去の履歴（コミット）に遡ると、まだRustのソースコードが見えてしまう状態**でございます。

Gitは「過去の状態」を全て記録しているため、単に今のファイルから消すだけでは、過去のコミットを表示することで中身を見ることができてしまいます。

## 解決策：履歴を1本にまとめて初期化する
GitHub上で一切のソースコードを遡らせないために、現在の「ソースを含まない最新の状態」を**最初で唯一のコミット**として履歴を書き換えます。

### 実施内容
1.  **新しい履歴の作成**: 現在の「綺麗な状態」から、過去の履歴を持たない新しいブランチ（孤立ブランチ）を作成します。
2.  **全ファイルの追加**: 実行に必要な最小限のファイル（現在の状態）のみをコミットします。
3.  **メインブランチの差し替え**: 過去の履歴を全て捨て、この「綺麗な1コミットだけの状態」を新しい `main` ブランチにします。
4.  **強制プッシュ**: GitHub上の履歴を、この新しい「ソースを一切含まない履歴」で上書きします。

### [!] ローカルファイルへの影響
- **ローカルの `rust_core` 等のファイルは一切削除しません**。そのまま保持されます。
- Gitの「記録（歴史）」だけが、ソースを含まないものに生まれ変わります。

## 実行予定のコマンド
```powershell
# 1. 履歴を持たない新しい枝を作成
git checkout --orphan latest_branch

# 2. 現在のファイルを全て追加
git add -A

# 3. 「初期コミット」として記録
git commit -am "Initial commit: Execution environment only"

# 4. 古い main ブランチを削除し、新しい枝を main に改名
git branch -D main
git branch -m main

# 5. GitHub に強制上書きプッシュ
git push -f origin main
```

## アーロン様へのご確認
この「履歴のクリーンアップ」を実施してもよろしいでしょうか。
これにより、GitHub上では過去に遡っても、Rustのソースコードは一切閲覧できなくなります。
(履歴が1つになり、本当に「全体で1本」の状態になります)
