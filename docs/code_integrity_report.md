# コード整合性維持に関する調査報告書

## 概要
Git履歴のクリーンアップに際し、ローカル環境のファイルとGit管理対象外となるファイルの不整合、およびWASMビルド成果物の依存関係を調査。

## 調査結果
1. **WASM依存**: pkg/ 配下のファイルは、JS側からスタティックにインポートされているため、Git管理対象に含めないと動作不能になる。
2. **パスの整合性**: index.html からの各資産（js, css, image）への相対パスは、ディレクトリ階層を変更しない限り維持される。
3. **ローカル環境の保護**: Gitの履歴消去コマンドが、インデックスにないローカルファイルを誤って削除しないことを dry-run で確認済み。

## 結論
構成を「実行環境のみ」に制限しても、アプリケーションの動作に影響はなく、かつソースコードの隠蔽という目的は達成可能である。
