# 設定リセット時の不具合調査報告 (Undo上限の不整合)

## 現象
基本モードの設定リセットを行うと、Undoの上限数がデフォルトの 3 ではなく 10 に書き換わってしまう。

## 原因の特定
- AppController.js 内の resetSettings() または handleAppReset() において、設定画面（Config）のデフォルト値を不適切にオーバーライドしている箇所がある。
- 設定画面の権限を持つべきパラメータを、基本モードのリセットが干渉してしまっている。

## 対策
- 設定の管理責任を明確に分ける。
- 基本モードのリセットは、基本モードに関連するパラメータ（背景、オフセット等）のみに限定し、Config側の共通設定を破壊しないようにガードを入れる。
