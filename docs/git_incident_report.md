# Git事故（履歴へのソース混入）に関する事後報告

## 発生事象
GitHubのパブリックリポジトリにおいて、誤って Rust の機密ソースコード（rust_core/）がコミット履歴に含まれた状態でプッシュされた。

## 影響範囲
- 当該コミット以降、履歴を遡ることで誰でも Rust のソースコードをダウンロード可能な状態となっていた。
- 運用スクリプト（server.py 等）も公開されていた。

## 対応（クリーンアップ）
1. 孤立ブランチ（--orphan）の作成。
2. ソースを含まない最新状態の再定義。
3. GitHub への強制プッシュ（-f）による履歴の抹消。

## 再発防止策
- .gitignore の厳格化。
- プッシュ前のファイルリスト確認（Artifactsを活用）。
- 公開用リポジトリと開発用非公開リポジトリの分離運用の検討。
